# План виправлень для узгодження з CONCEPT_LOCK

1. **agentfw/web/static/index.html** — прибрати бічний інспектор та додати на канву три області змінних поточного агента (inputs зліва, locals зверху, outputs справа) з місцями для інтерактивних прямокутників. Межі: правки тільки структури DOM у цьому файлі.

2. **agentfw/web/static/styles.css** — оформити нові області змінних на краях канви та вирівняти порти карток/лінії з’єднань під новий макет; без зміни поведінки JS. Межі: лише стилі, без торкання логіки.

3. **agentfw/web/static/app.js** — реалізувати додавання/редагування/видалення змінних поточного агента на канві, відображення їх як окремих портів на краях, і drag&drop з’єднань між портами поточного агента та портами дочірніх агентів (inputs/locals/outputs). Межі: тільки JS-логіка фронтенду.

4. **agentfw/runtime/engine.py** — перевірити/підлаштувати застосування bindings з/до контексту поточного агента відповідно до нових портів, не змінюючи зовнішній контракт API. Межі: лише логіка виконання та обробки bindings.

5. **agents/workflow_demo.yaml** — оновити демо під нову схему портів (вхід/локальні/вихідні агента, явні bindings) і використати атомарний агент розгалуження. Межі: тільки цей YAML.

6. **tests/** — додати/оновити тести для: drag-з’єднань контекст→input/output→context, відображення змінних поточного агента, та роботу демо workflow через API. Межі: лише файли тестів.
