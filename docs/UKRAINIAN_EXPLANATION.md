# Пояснення, чому агент повертає стан blocked

Цей стан означає, що Chat-Agent очікує реальне повідомлення користувача, а текст у запиті був порожнім або складався лише з пробілів.

- **Що відбувається.** Адаптивний агент одразу передає керування `chat_agent`, який показує запитання користувачу й переводить виконання у режим очікування. Він не може продовжити без змістовного вводу, тож повертає статус `blocked` із запитанням у полі `questions_to_user`.
- **Чому без сесій чи автозапусків.** Чат працює як один процесний канал: історія зберігається тільки в пам'яті, жодних сесій чи стартових подій немає. Якщо приходить порожнє повідомлення, Chat-Agent його відхиляє та лишається в режимі очікування.
- **Як діяти.** Надішліть звичайне текстове повідомлення (опис завдання або відповідь на уточнення) через чат. Після цього агент отримає текст, зніме блокування і продовжить виконання плану.

Ця поведінка відповідає концепції agent-first: агент ставить питання, а без відповіді користувача нічого не робить і не запускається самостійно.
